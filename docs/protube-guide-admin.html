User guide

A quiet way to save what you watch.

ProTube Saver is a Windows app for downloading YouTube videos and playlists locally — built around a queue you can return to, a library that survives offline, and a player that just works. This guide walks through everything you need to use it well.

Reading time · ~8 min Platform · Windows 10/11 Type · Portable .exe
01

What ProTube Saver is

It's a desktop YouTube downloader. You paste a link, it downloads. Pretty much the whole pitch.

Underneath, it's a small Python app wrapping yt-dlp with a custom UI, a download queue, a local library, and a built-in video player. Designed for people who want their stuff on disk — for travel, for archiving, for offline viewing — without dealing with command lines or sketchy websites.

A few things worth knowing up front:

  • Portable. The exe lives in a folder of your choosing. All settings, library data, and downloads sit next to it in a data/ folder. Move the folder, everything moves.
  • Offline-first. Once a video is downloaded, you can play it inside the app even without internet. Thumbnails are cached locally too.
  • No accounts. No sign-in, no telemetry, no cloud anything.

Install & first launch

There's no installer. Just an exe.

  1. Drop ProTubeSaver.exe in any folder you'd like the app to live. C:\Apps\ProTube\ works. So does a USB stick.
  2. Double-click the exe.
  3. On first launch, Windows SmartScreen may show a warning (unsigned binary). Click More info → Run anyway.
  4. The app opens. A data/ folder appears next to the exe — that's where your settings and library will live from here on.
Note

If you want to move the app later, close it first, then move the whole folder (exe + data/ together). Everything follows.

03

Adding videos to the queue

Three ways. All of them work on single videos and full playlists.

Paste a URL

The fastest way. Click the URL bar at the top, paste a YouTube link, press Enter. The app fetches metadata (title, channel, duration, available qualities) and adds it to the queue. Playlist URLs add as a single playlist card you can expand.

Drag & drop

Drag a YouTube link from your browser tab directly into the app window. A blue drop overlay appears — release the link, same as paste.

Multiple at once

Paste multiple URLs separated by newlines or spaces. Each becomes its own queue item. Useful when you've copied a batch of links from a doc or chat.

Tip

Default download quality is whatever you last picked. If you want a different quality for one video, use the per-item picker (next section) — your default doesn't change.

04

Picking quality

Per-video, per-playlist, or globally.

Each queue item has a quality picker showing the formats YouTube exposes for that video — usually 4K, 1440p, 1080p, 720p, 480p, plus audio-only. Click the picker, choose, done.

For playlists, the picker on the playlist card sets the default for all videos inside. Individual videos in the playlist can still override.

The picker is smart about positioning — if there's no room below (last card in the queue), it flips and opens upward. Either way, your layout doesn't shift.

Heads up

Higher quality = larger file. 4K can run 1.5–3 GB for a 10-minute video. Audio-only is best for podcasts or music.

05

Managing the queue

The queue is the heart of the app. Here's how to bend it.

Pause & resume

While a download is running, hover the card and hit pause. The download stops mid-stream and writes a .part file to disk. Hit resume later — even after closing and reopening the app — and it picks up exactly where it left off.

Cancel & remove

Cancel removes the item from the queue. If it was mid-download, the partial file is cleaned up. Use the kebab menu (the ⋯ on each card) for full options including "Open folder" and "Copy URL".

Reorder

Drag any queue card up or down to change download order. Useful when you've added a long playlist but want to grab one specific video first.

Selection mode

Right-click any card (or use the per-card checkbox on hover) to enter selection mode. From there, every click toggles selection. An action bar slides up from the bottom with bulk options. Esc exits selection mode without acting.

06

Library mode

Everything you've ever downloaded, in one place.

Once a download finishes, it moves to the Library — accessible from the left rail. Library is grid-based by default: thumbnail, title, channel, duration. Click any card to open the in-app player.

Library entries persist across sessions and survive offline. Thumbnails are cached locally on first download, so even with no internet your grid stays populated.

Search & filter

The search bar matches against title and channel. Filters let you narrow to playlists, audio-only, or specific resolutions.

Playlists

Downloaded playlists show as a single card with a count badge. Click to expand into the playlist detail view, where you can play individual videos, re-order, or remove from library.

07

The in-app player

YouTube-style controls, full keyboard support, fullscreen that actually works.

Double-click any library card and the player view opens. Top overlay shows the title and a back button. Bottom overlay has the standard cluster: play/pause, skip, volume, time, playback speed, fullscreen.

Controls auto-hide after 2 seconds of no movement. Move the mouse and they fade back in. Click anywhere on the video itself to play/pause.

08

Settings

There aren't many. That's intentional.

Download folder

By default, downloads go into <app folder>/data/downloads/. To change it: click the folder path at the bottom of the queue view, pick a new location. Existing files stay where they are. New downloads write to the new path.

Concurrent downloads

The app downloads up to 2 videos in parallel by default. This is a balance between speed and not getting rate-limited by YouTube. Coming soon: a slider to tune this.

yt-dlp updates

The app auto-updates its bundled yt-dlp on launch in the background. YouTube changes their internals fairly often; without updates, downloads start failing within weeks. You don't have to do anything — it just runs.

When things go wrong

Common issues and how to unstick them.

"Video unavailable" or fetch fails

Usually means YouTube changed something and yt-dlp hasn't caught up yet. Wait a few hours and try again — the auto-updater will pull a fix on next launch. If urgent, reinstall yt-dlp manually (advanced).

Download stuck at 99%

The merge step (combining video + audio streams via ffmpeg) is running. Give it 10–30 seconds. If it stays stuck for over a minute, cancel and retry the download — the partial file will be cleaned up automatically.

Quality I want isn't shown

YouTube doesn't expose every resolution for every video. If 1440p is missing, it likely doesn't exist for that upload. The picker shows what's actually available, not what's theoretically possible.

App won't launch on a new machine

Make sure WebView2 Runtime is installed. Most Windows 10/11 systems have it preinstalled, but some stripped builds don't. Download the runtime from Microsoft's site, install, retry.

Lost my settings / library

Check next to your exe for a data/ folder. If it's gone, restore from backup if you have one. The app falls back to a fresh install when data/ is missing — your downloaded video files are still on disk, just not indexed in the library anymore.

Last resort

Close the app. Delete data/settings.json only (keep the folder and other files). Relaunch. The app rebuilds with defaults but keeps your library.
